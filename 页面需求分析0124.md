## Use Case 1：信用卡智能預審
展示：
ASR
任務型對話
風控前置篩選
CRM / Case 建立（Mock）
UI 問題流程（外包必須照這個順序）
Step 1｜啟動
UI 顯示
👉「我可以幫你快速評估信用卡申請資格」
Step 2｜核心問題（最多 3 題）
「請問你的年齡是否介於 20–65 歲？」
「是否目前有穩定收入？」
「是否持有本地身份證件？」
UI：是 / 否 按鈕
ASR：可以語音回答
Step 3｜結果提示
「根據你提供的資料，你符合初步申請條件」
Step 4｜下一步選擇
「是否需要我幫你建立申請紀錄，並由專人跟進？」
👉 這一步要 產出 Mock API Call
建立 PreApprovalCaseID
顯示成功畫面

## Use Case 2：開戶 / 分行預約
展示：
STM 作為「前台分流器」
不碰核心系統也能有價值
UI 問題流程
Step 1｜啟動
「我可以協助你預約開戶或分行服務」
Step 2｜選擇服務類型
開戶
貸款諮詢
財富管理
其他

Step 3｜選擇時間
UI 顯示 3 個可選時段（Mock）
Step 4｜確認
「我已幫你建立預約，請於指定時間到分行」
👉 外包必須做：
預約確認畫面
Mock 預約成功回傳

## Use Case 3：銀行智能服務問答
展示：
LLM + RAG
多語言能力
STM 作為「AI 櫃員」
UI 問題範例（限定型）
「開戶需要準備哪些文件？」
「分行營業時間是？」
「信用卡年費如何計算？」
關鍵要求：
回答必須來自 固定知識庫
不允許自由發揮
👉 只需：
FAQ JSON
HTML5 UI
Azure TTS 播放
