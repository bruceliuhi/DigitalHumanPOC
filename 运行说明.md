# 数字人应用 (DigitalHumanPOC) 运行说明

文档更新时间：2026-02-01

本项目包含前后端两个主要服务，需要同时启动才能完整体验数字人功能。

## 1. 项目目录结构

主要目录位于 `POCDEMO` 文件夹下：

*   **前端项目 (DigitalAssistantInterface_clean)**:
    *   React + Vite 框架
    *   包含数字人展示、银行卡业务 UI、响应式布局
    *   **启动后端口**: `3000` (或其他 Vite 自动分配端口)
*   **后端服务 (DEMO/server)**:
    *   Node.js 服务
    *   负责处理 Web Socket 连接、ASR（语音识别）模拟转发、业务逻辑编排
    *   **服务端口**: `18080`

---

## 2. 快速启动指南

### 第一步：启动后端服务 (Backend)

后端服务负责与数字人 SDK 通信及处理业务逻辑。

1.  打开一个新的终端窗口 (Terminal)。
2.  进入 server 目录：
    ```powershell
    cd c:\Users\bruce\AI\DigitalHumanPOC\POCDEMO\DEMO\server
    ```
3.  安装依赖 (仅首次需要)：
    ```powershell
    npm install
    ```
4.  启动服务：
    ```powershell
    node src/index.js
    ```
    *启动成功通过检查：* 终端输出 `🚀 Server running on http://localhost:18080`

### 第二步：启动前端应用 (Frontend)

前端应用是用户交互界面。

1.  打开另一个新的终端窗口。
2.  进入前端项目目录：
    ```powershell
    cd c:\Users\bruce\AI\DigitalHumanPOC\POCDEMO\DigitalAssistantInterface_clean
    ```
3.  安装依赖 (仅首次需要)：
    ```powershell
    npm install
    ```
4.  启动开发服务器：
    ```powershell
    npm run dev
    ```
    *启动成功通过检查：* 终端显示访问地址，通常为 `http://localhost:3000/`

## 3. 访问与验证

1.  打开浏览器（推荐 Chrome 或 Edge）。
2.  访问地址：**[http://localhost:3000](http://localhost:3000)**
3.  **验证点**：
    *   页面应显示黑色背景或 UI 界面。
    *   如果不报错且能看到界面元素，说明前端运行正常。
    *   如果需要与数字人交互，请确保后端服务的 WebSocket (`ws://localhost:18080`) 连接正常（通常在控制台 Network 面板可查看 WS 连接）。

## 4. 常见问题排查

*   **Q: 端口被占用？**
    *   如果 `3000` 端口被占用，Vite 会自动切换到下一个可用端口（如 3001）。请留意终端输出的实际地址。
    *   如果后端 `18080` 被占用，请检查是否有旧的 `node` 进程在运行，可以尝试手动关闭或重启电脑。
*   **Q: 安装依赖报错？**
    *   请确保网络连接正常，可以尝试删除 `node_modules` 文件夹后重新 `npm install`。
*   **Q: 页面显示空白？**
    *   按 `F12` 打开开发者工具，查看 Console 面板是否有红色报错信息。
