# 数字人接口 (DigitalAssistantInterface) 运行说明

## 1. 项目位置
本项目位于：`/Volumes/Users/bruce/AI/DigitalHumanPOC/POCDEMO/DigitalAssistantInterface`

## 2. 环境要求
- Node.js (推荐 v18 或更高版本)
- npm (通常随 Node.js 安装)

## 3. 首次运行步骤

### 3.1 进入项目目录
打开终端，进入项目目录：
```bash
cd /Volumes/Users/bruce/AI/DigitalHumanPOC/POCDEMO/DigitalAssistantInterface
```

### 3.2 安装依赖
如果您尚未安装依赖，或者遇到依赖错误，请运行：
```bash
npm install
# 如果遇到权限错误，尝试清理缓存后重试
rm -rf node_modules package-lock.json
npm install
```
*注意：由于本项目添加了 `tcplayer.js` 等新依赖，请确保 `npm install` 成功完成。*

### 3.3 启动开发服务器
```bash
npm run dev
```
启动成功后，终端会显示本地访问地址，通常为：`http://localhost:5173/`

## 4. 访问数字人界面
主页可能显示默认内容，数字人专属界面在：
**地址**: `http://localhost:5173/digital-assistant`

## 5. 常见问题
- **依赖安装权限错误 (EACCES)**: 请尝试使用 `sudo npm install` 或清理 `node_modules` 后重试。
- **页面空白**: 请检查浏览器控制台 (F12) 是否有报错。
- **数字人未显示**: 确保网络正常，且能连接到数字人 SDK 所需的云端服务。如果本地有 Mock Server 要求，请确保 `DEMO/server` (如有) 也已启动。
- **WebSocket 断开**: 界面显示 "断开" 表示无法连接到 `ws://localhost:8080`。这是 `DEMO` 中的后端服务，请参照 `DEMO` 文档启动该服务以获得完整交互体验。
