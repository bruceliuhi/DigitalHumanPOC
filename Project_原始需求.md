# 数字人 STM 项目整体需求文档

> Digital Human POC - 整体需求与规划

---

## 📌 项目概述

1 系统总体功能范围
本项目为 Demo/PoC 级演示系统，基于 HTML5 技术栈构建，核心目标为演示“数字人+语音交互+人机协同+风险与合规流程”的整体能力，不作为生产系统或真实业务系统使用。

系统整体功能包括：
•	硬件 ASR 语音输入接入（文本级）
•	Azure TTS 语音合成输出
•	2D 数字人 SDK 驱动与播报
•	Integration Layer（集成编排层）
•	人机协同与风险分级演示逻辑
•	决策流程可视化与 Mock 审计日志展示
2.语言支持范围
1.	系统支持以下语言用于 Demo 演示：
o	中文普通话
o	英文
o	粤语
2.	语言能力说明：
o	语音识别（ASR）与语音合成（TTS）均基于第三方能力实现；
o	Demo 场景下支持使用上述语言进行语音交互演示；
o	不包含多语言自动切换、语种智能识别优化或模型训练与调优。
3.UI 与交互功能范围
1.	UI 形态说明：
o	系统 UI 以功能性展示为主，不包含视觉设计、品牌定制或复杂交互动效；
o	数字人展示基于现有 2D 数字人 SDK 实现。
2.	建议问题展示功能：
o	在数字人 UI 版面中固定展示 三个建议问题入口；
o	每个建议问题以“图标 + 文本”形式呈现，用于示例引导；
o	建议问题内容为预设示例，可预先配置但不支持后台动态管理；
o	用户点击建议问题后，系统将其作为用户输入进入正常对话流程。
4.人机协同与风险控制演示范围
1.	人机协同（Human-in-the-loop）演示：
o	AI角色定位为辅助；
o	人工角色定位为决策者；
2.	风险分级逻辑（静态规则）：
o	Low：信息查询类问题；
o	Medium：流程指引类问题；
o	High：涉及金钱、法律、投资或身份等敏感内容。
3.	Demo 演示要求：
o	展示一次风险等级升级过程（如 Low → Medium）；
o	UI 或侧边区域显示风险等级变化；
o	支持一次完整的“转人工”流程演示，并显示“转人工中（Demo）”状态。
5. 决策流程与审计日志（Mock）
1.	决策流程可视化：
o	展示用户输入、风险判断、AI 响应/升级、人工接管等节点；
o	可在 Demo 演示中完整讲解流程，部分背后逻辑为模拟实现。
2.	Mock 审计日志展示：
o	展示字段包括：Session ID、时间、风险等级、是否转人工、对话摘要；
o	支持以表格或JSON形式展示数据结构；
o	不接入真实业务数据，仅用于结构与流程说明。
6. 范围与边界说明
1.	本项目不包含：
o	生产级系统架构、安全加固与运维能力；
o	ASR/TTS 模型训练、调优或算法优化；
o	真实人工坐席系统、真实银行系统或真实数据对接；
2.	如项目范围发生变化或需升级为生产级系统，双方需另行签署补充协议并确认费用。
3.	本合同仅适用于 Demo / PoC 级演示系统，不包含生产系统级稳定性、性能优化、安全加固与运维能力。
4.	项目中所有业务流程、审计日志、风险判断均为示例逻辑，不接入真实银行系统、真实用户数据或真实决策系统。
5.	UI 不进行视觉设计，仅基于数字人 SDK 与结构化界面进行 Demo 展示。
6.	若后续需求范围发生变化，双方需另行确认工作内容与费用。
